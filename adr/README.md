# 📁 adr/
>Architecture Decision Records
Ce dossier contient les Architecture Decision Records (ADR) du projet github_basic.  
Les ADR documentent chaque décision technique importante, son contexte, les alternatives envisagées et la justification finale.

Ils permettent de :

- Garder une traçabilité claire des choix d’architecture  
- Comprendre l’évolution du projet dans le temps  
- Faciliter l’onboarding des nouveaux contributeurs  
- Assurer une cohérence technique dans les futures versions  
- Documenter les impacts, risques et motivations des décisions

---

🧩 Structure des ADR

Chaque fichier ADR suit un format standardisé :

1. Titre  
2. Contexte  
3. Problème / Besoin  
4. Options envisagées  
5. Décision  
6. Conséquences  
7. Références

Les fichiers sont nommés ainsi :

`
adr/
├─ 0001-titre-de-la-decision.md
├─ 0002-choix-du-langage.md
└─ 0003-structure-du-projet.md
`

---

🛠️ Outils recommandés

Pour maintenir les ADR :

- Format Markdown simple  
- Numérotation incrémentale  
- Un fichier par décision  
- Historisation via Git pour suivre les évolutions  

---

📘 Exemple de modèle ADR

Tu peux utiliser ce template pour créer de nouveaux ADR :

`markdown

ADR X — Titre de la décision

Contexte
Décrire le contexte technique, fonctionnel ou organisationnel ayant mené à cette décision.

Problème
Quel problème ou besoin fallait-il résoudre ?

Options envisagées
- Option A
- Option B
- Option C

Décision
Décrire la solution retenue et pourquoi elle a été choisie.

Conséquences
- Conséquence positive
- Conséquence négative
- Impact sur le projet

Références
Liens, documents, discussions, issues GitHub…
`

---

🚀 Objectif du dossier

Ce dossier garantit que toutes les décisions techniques majeures du projet github_basic sont :

- Documentées  
- Compréhensibles  
- Justifiées  
- Traçables  
- Pérennes  

Il constitue une mémoire technique essentielle pour la maintenance et l’évolution du projet.
